git

安装

  • https://git-for-windows.github.io:windows上的安装
  • git config --global user.name "Your Name":设置名字
  • git config --global user.email "email@example.com":设置邮箱
  • git config --list:查看参数设置对了没

创建版本库

  • mkdir 目录名
  • cd 目录名
  • git init:把这个目录变成Git可以管理的仓库。
  • git add readme.txt:添加制定文件readme.txt
  • git add .:添加所有文件
  • git commit -m "":提交

第一次提交后，可以直接使用git commit -a -m "提交说明"进行一次性提交，而不用走git add这一步，不过默认会提交所有文件；

时光机穿梭

回退版本

  • git status:查看状态
  • git diff:查看不同
  • git log --pretty=oneline:查看提交历史
  • git reflog:查看命令历史，以便确定要回到未来的哪个版本。
  • git reset --hard HEAD^:回退到上一个版本
  • git reset HEAD/git reset HEAD file：撤销提交
  • git reset --hard commitID:回退到指定版本
  • 工作区与暂存区的概念
  • git checkout -- readme.txt:把readme.txt文件在工作区的修改全部撤销

删除文件

  • rm test.txt:删除工作区文件
  • git rm test.txt:从版本库中删除

添加到远程仓库

第一步
在github创建一个项目
第二步
创建本地与远程仓库之间的关联，git remote add origin 你的git地址
第三步
git push -u origin master
注意如果push出现问题解决办法是

1、git pull;先拉回来

2、git pull origin master --allow-unrelated-histories允许合并不相关历史

从远程仓库克隆，只需要一句话git clone 你的git仓库地址

分支管理

创建

git checkout -b dev
创建并切换到分支dev
git branch dev / git checkout dev
创建分支 /切换分支
git branch
查看当前分支

合并

  • git merge dev:当前分支合并分支dev

删除分支

  • git branch -d dev:删除dev分支
  • git branch -D dev:强行删除没有合并过的分支

bug分支

  • git stash:把当前工作现场“储藏”起来，等以后恢复现场后继续工作
  • git stash list:查看存储历史
  • git stash apply stash@{0}:恢复指定储藏版本
  • git stash pop:恢复的同时把stash内容也删了

多人协作

  • git push origin dev
  • git pull:出现问题时执行git branch --set-upstream dev origin/dev:push
